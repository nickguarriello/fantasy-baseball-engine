# Fantasy Baseball H2H Decision Engine

A Python pipeline that fetches live MLB stats, ranks every player by z-score,
and generates actionable waiver wire, matchup, and trade recommendations for
ESPN H2H category fantasy baseball leagues.

---

## Quick Start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Set your league info in config.py
#    LEAGUE_CONFIG = { 'league_id': ..., 'team_id': ..., 'season': 2026 }

# 3. Run the pipeline (Windows — -X utf8 required for emoji output)
python -X utf8 main.py

# 4. Open the dashboard UI
streamlit run app.py
```

Runtime: ~30–60 seconds (7 MLB API calls + ESPN call).
The UI reads from the CSVs generated by the pipeline — always run `main.py` first.

`app.py` is a Streamlit web app with 5 tabs:

## What It Does

### Phase 1 — Player Rankings
Fetches full-season stats for every MLB player who has appeared this year
and calculates z-scores per position group.

**Z-score windows:**
| Column | What it means |
|--------|--------------|
| `z_season` | Full season (PRIMARY — always use this to rank) |
| `z_7day` | Last 7 days — hot/cold streak signal (noisy early season) |
| `z_14day` | Last 14 days |
| `z_30day` | Last 30 days |

All outfield sub-positions (CF, LF, RF) are consolidated to **OF**.

### Phase 2 — Waiver Wire
- Ranks all unrostered MLB players by `z_season`
- Flags **two-start pitchers** available on waivers (huge weekly value)
- Filters out injured/IL players from recommendations
- Identifies your **weakest H2H categories** and surfaces top pickups for each
- Position-filtered CSVs (`waiver_of.csv`, `waiver_ss.csv`, etc.)

### Phase 3 — Matchup Analysis & Lineup
- Category-by-category z-score comparison vs your current opponent
- **Actual vs projected**: ESPN in-week accumulated stats alongside projections
  (lets you track if your decisions are working — divergence flags highlight surprises)
- Start/sit recommendations with:
  - Injured players automatically flagged as DO NOT START
  - Two-start pitchers boosted in sort order (clearly labelled)
  - Matchup-need boost: borderline players elevated if they help in categories you're losing

### Phase 4 — Trade Analysis
- Scans all opponent rosters for high-z players worth targeting
- Lists your trade chips (positive z-score players)
- Shows best player per opponent team
- Specific trade evaluation via `--trade` flag

## Usage Examples

```bash
# Skip phases you don't need this run
python -X utf8 main.py --skip-phases waiver trade

# Evaluate a specific trade offer
python -X utf8 main.py --trade "Nico Hoerner,Mason Miller" "Drake Baldwin"

python -X utf8 main.py --skip-phases waiver matchup trade
```

## How Z-Scores Work

Z-score = `(player_stat - position_average) / standard_deviation`

- **+2.0** = elite (top ~2% of position)
- **0.0** = exactly average
- **-2.0** = poor (bottom ~2%)

Computed separately for hitters and pitchers, within position groups (C, SS, OF, SP, RP, etc.).
For lower-is-better stats (ERA, WHIP), the z-score is inverted so higher is always better.

## Data Sources

- **MLB Stats API** (`statsapi.mlb.com`) — no API key needed
- **ESPN Fantasy API** (`lm-api-reads.fantasy.espn.com`) — public league, no login needed

Free agents are derived by cross-referencing the MLB stats pool against all ESPN
rostered players (name matching). ESPN's free agent endpoint requires authentication
even on public leagues, so we compute it instead.
